## 旅享 Travel - 轻计划旅游官网模板

**本项目由阿里云ESA提供加速、计算和保护。**

> 说明：本项目用于参加“阿里云边缘安全加速ESA · Pages 边缘应用开发大赛”，已按要求适配在 ESA Pages 上的静态托管与前端加速场景。

---

### 一、项目介绍

`旅享 Travel` 是一个基于 **Vue 3 + Vite** 构建的旅游类官网模板，适合作为旅行社、小团定制游、目的地内容站的落地页或完整官网使用，特点是**开箱即用、易于二次开发、适配边缘加速场景**。

- **实用性**
  - 首页包含 Hero 区、热门目的地、精选线路、旅行攻略与行动召唤按钮，可直接作为投放落地页使用。
  - 目的地、线路、攻略、联系等栏目结构完整，表单字段贴合真实业务，轻改文案与数据即可用于生产。
  - 所有文案、货币单位等面向中文场景，方便国内业务直接使用。

- **创意性**
  - 采用“**轻计划旅行**”理念，聚焦周末/短假期、轻徒步、亲子等高频出行场景，通过文案与布局引导用户快速决策。
  - 设计上使用暖色渐变 + 卡片化栅格布局，突出目的地灵感卡片与小团线路卖点，兼顾美观与转化。
  - 404 页面、当季推荐、灵感攻略等细节，增强整体体验的完整度和品牌感。

- **技术深度**
  - 前端采用 **Vue 3 组合式 API + Vue Router + Pinia** 的标准现代架构，方便扩展为完整 SPA 或接入后端 API。
  - 以 **配置式数据 store** 管理目的地、线路与攻略数据，方便后续接入边缘存储 / KV / API 网关等。
  - 经过样式抽象的 UI 体系（按钮、卡片、标签、栅格、表单），适合作为 ESA Pages 上构建多站点的基础模板。

### 二、功能概览

- **首页（/）**
  - Hero 大图区：品牌标语、核心卖点、行动按钮（查看线路/发现目的地）
  - 当季推荐：从热门目的地中挑选推荐卡片展示
  - 热门目的地：展示多张目的地卡片（封面图、地区标签、卖点文案）
  - 精选线路：展示轻团行程卡片（天数、亮点、价格区间等）
  - 旅行灵感：展示多篇攻略摘要卡片

- **目的地页（/destinations）**
  - 使用卡片栅格展示所有目的地，包含地区标签、卖点标签、热门标识等。

- **线路页（/tours）**
  - 按卡片列表展示精选线路，展示天数、亮点、价格起步与咨询入口。

- **攻略页（/blog）**
  - 展示预置的多篇旅行攻略文章摘要及标签，可扩展为完整博客系统。

- **联系页（/contact）**
  - 收集用户：称呼、目的地、时间与人数、预算、联系方式等核心信息的表单。
  - 同时提供电话/邮箱/微信等多种联系入口，贴合真实业务需求。

- **404 页面**
  - 简洁友好的 404 提示与导航引导，保证整体体验闭环。

### 三、技术栈与架构

- **前端框架**：Vue 3（组合式 API）
- **构建工具**：Vite
- **路由**：Vue Router 4
- **状态管理**：Pinia
- **样式**：原子化设计的全局 CSS（无第三方 UI 依赖，方便在边缘环境中精简体积）

项目主体结构：

- `src/main.js`：应用入口，挂载 Vue，接入 Pinia 与 Router。
- `src/router/index.js`：路由配置与页面标题管理。
- `src/stores/site.js`：站点基础数据（品牌信息、目的地、线路、攻略等）。
- `src/App.vue`：整体布局，包含统一头部与底部。
- `src/components/layout/*`：头部与底部布局组件。
- `src/views/*`：按路由划分的页面视图组件。
- `src/style.css`：全局样式与基础 UI 组件（按钮、卡片、标签、栅格、表单等）。

### 四、本地开发与构建

#### 1. 本地运行

```bash
npm install
npm run dev
```

启动后按照 Vite 输出的本地地址访问（默认为 `http://localhost:5173`）。

#### 2. 生产构建

```bash
npm run build
```

构建产物默认输出到 `dist/` 目录，可直接用于静态托管与 ESA Pages 部署。

### 五、在阿里云 ESA Pages 上部署

> 以下为在 ESA Pages 平台上的标准部署流程示例，实际操作以 ESA Pages 控制台最新文档为准。

1. **准备 GitHub 仓库**
   - 将本项目推送至你的 **公开 GitHub 仓库**。
2. **在 ESA Pages 创建站点**
   - 登录阿里云控制台，在“边缘安全加速 ESA”中进入 **Pages**。
   - 创建新站点，选择“从 Git 仓库导入”，绑定你的 GitHub 账号并选择本项目仓库。
3. **构建配置**
   - 构建命令：`npm install && npm run build`
   - 构建输出目录：`dist`
   - Node 版本：使用 ESA 默认或指定 LTS 版本（例如 Node 20），以避开 Vite engine warning。
4. **域名与访问**
   - 部署完成后，ESA Pages 会分配一个默认访问域名。
   - 如需要，可在控制台绑定自定义域名，并开启 HTTPS 与缓存策略。
5. （可选）**结合 ESA 其他能力**
   - 可进一步结合边缘函数对特定路径进行动态渲染、A/B 测试或请求重写。
   - 使用边缘存储（如 KV/对象存储）持久化目的地与线路数据，实现无后端的动态内容更新。

### 六、作品说明（评审维度）

> 下述文案可直接用于提交 TXT 文件中的“作品说明”部分，也可按实际情况微调。

1. **创意卓越**
   - 围绕“轻计划旅行”这一主题，站点设计从品牌标语、排版到配色都服务于“周末/短假期快速决策”这一核心场景。
   - 首页通过当季推荐、热门目的地卡片与精选线路模块串联完整决策链路，既能激发想法，又能快速转化为咨询。
   - 使用细腻的卡片化视觉与渐变背景，兼顾移动端与桌面端展示效果，突出目的地灵感与品牌感。

2. **应用价值**
   - 模板结构贴合真实旅游业务：目的地库 + 小团行程 + 攻略内容 + 咨询入口，部署即用，只需调整数据与品牌信息。
   - 联系页面的字段设置基于实际运营经验，能帮助前线顾问在低成本沟通下获取完整关键信息，提高转化效率。
   - 前端完全静态化 + ESA 边缘加速，非常适合作为投放落地页、多渠道活动页或独立品牌官网的基础模版。

3. **技术探索**
   - 使用 Vue 3 + Vite + Vue Router + Pinia 构建现代 SPA 架构，清晰划分布局组件、视图组件与数据 store。

   - 前端样式不依赖第三方 UI 库，以自定义轻量 CSS 实现组件化视觉，保证在边缘环境中的首屏性能与可控体积。
